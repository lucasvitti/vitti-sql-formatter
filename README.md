# Vitti SQL Formatter

Tools that reformat any SQL into **your** indentation style (the "Vitti style" —
river layout, leading commas, `where 1=1`, aligned aliases), reverse-engineered
from 47 real-world SQL scripts.

Deliverables:

| # | What | Where |
|---|------|-------|
| 1 | **Web formatter** (paste → format) | `web/` → live at https://identar.lucas.mat.br |
| 2 | **Chrome/Edge extension** for Databricks-in-browser | `extension/` |
| 3 | **Human-readable rules** | [`RULES.md`](RULES.md) |
| 4 | **HTTP API** (programmatic) | `api/` → live at https://identar.lucas.mat.br/api/ ([`api/README.md`](api/README.md)) |

Everything shares one engine: **`sqlfmt/sqlfmt.js`** (zero dependencies; runs in
Node, the browser, the extension, and the API).

---

## The engine — `sqlfmt/sqlfmt.js`

```js
const SQLFmt = require('./sqlfmt/sqlfmt.js');
SQLFmt.format(sql, { keywordCase:'lower', useTabs:true, tabWidth:4, where11:true });
```

Options (defaults shown): `tabWidth:4`, `riverWidth:12`, `useTabs:true`,
`keywordCase:'lower'`, `functionCase:'lower'`, `where11:true`,
`blankBetweenClauses:true`, `alignAliases:true`, `caseMultiline:true`,
`overMultiline:true`.

Iterate / test:
```
node sqlfmt/test.js          # core cases
node sqlfmt/test2.js         # harder shapes (subquery, union, sum(case), delete)
```

`web/sqlfmt.js` and `extension/sqlfmt.js` are **copies** of the canonical
`sqlfmt/sqlfmt.js`. After editing the engine, re-sync:
```
sync-engine.cmd              # copies sqlfmt/sqlfmt.js into web/ and extension/
```

### Scope
Targets query DML you actually write on Databricks — `select` / `insert…select` /
`update…from` / `delete`, CTEs, `case`, window functions, subqueries. Procedural
T-SQL wrappers (`create/alter procedure`, `declare`, `exec`, `if`, `begin…end`,
`use`, `go`) are **passed through untouched**, so pasting a whole stored procedure
reformats the embedded queries and leaves the scaffolding alone. Known v1 edge:
`case` nested inside an aggregate (`sum(case…)`) stays on one line; T-SQL table
hints like `(nolock)` lose their space.

## 1. Web formatter (`web/`)

Static page, no build step. Four **alignment rulers** — `fields` (operand river),
`labels` (select aliases; `0`=auto-fit), `joins` (align after `=` in JOIN `on`/`and`;
`0`=off), `filter` (align after the comparison in `WHERE`/`HAVING`; `0`=off) — each a
number box plus a draggable, color-coded guide line over the output. Type or drag to
move; the SQL re-aligns live. Try it locally:
```
python -m http.server 8137 --directory web
# open http://127.0.0.1:8137
```

**Live at https://identar.lucas.mat.br** (deployed 2026-06-25). To re-deploy after editing
`web/` (no local rsync → tar-over-ssh, `vps` = the Hostinger VPS):
```
tar -C web -czf - index.html sqlfmt.js RULES.html \
  | ssh vps "tar xzf - -C /var/www/identar.lucas.mat.br && chown -R www-data:www-data /var/www/identar.lucas.mat.br"
```
nginx vhost: `/etc/nginx/sites-available/identar.lucas.mat.br` (HTTP→HTTPS redirect, LE cert via
certbot `--nginx`, auto-renew). Static files only — just re-push and the change is live (no reload needed).

## 2. Browser extension (`extension/`)

Manifest V3, works in Chrome and Edge. Formats 100% locally (no network calls), and is
submitted to the Chrome Web Store and Microsoft Edge Add-ons.

**Install (unpacked):**
1. `chrome://extensions` (or `edge://extensions`) → enable **Developer mode**.
2. **Load unpacked** → select the `extension/` folder.

**Use it:** select your SQL and press **Ctrl+Shift+Y** (or right-click → **"Format SQL →
Vitti style"**) to replace the selection in place. Column spacing and other options live
in the popup's **Config** tab.

**On Databricks:** the cell editor is Monaco, which doesn't expose its text to the page,
so use a clipboard round-trip:
1. Select the SQL → **Ctrl+C**.
2. Click the toolbar icon (the popup auto-formats the clipboard) **or** press **Ctrl+Shift+Y**.
3. **Ctrl+V** to paste the formatted SQL back.

Icons are generated by `extension/gen-icons.js` (`node gen-icons.js`).
